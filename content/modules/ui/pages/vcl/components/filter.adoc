Компонент `Filter` – универсальное средство фильтрации данных.

Компонент позволяет производить <<quick-filtering,быструю фильтрацию данных>> по произвольному набору условий, а также создавать <<configuration,фильтры>> для многократного использования.

XML-имя компонента: `filter`.

== Основы

`Filter` должен быть связан с загрузчиком данных независимого `CollectionContainer` или `KeyValueCollectionContainer`. Он генерирует объект `Condition`, который устанавливается в загрузчик и затем обрабатывается xref:data-model:data-stores.adoc[хранилищем данных]. Для xref:data-model:entities.adoc#jpa[JPA-сущности] хранилище данных изменяет результирующий запрос JPQL. Таким образом, фильтрация осуществляется на уровне БД, и в память приложения загружаются только отобранные данные.

По умолчанию Jmix Studio генерирует `Filter` при создании для сущности экранов браузера и master-detail.

+

[[add-condition]]
== Добавление условий

Для того чтобы создать быстрый фильтр, нажмите на ссылку *Add search condition* (<<add-condition-link,1>>). Отобразится экран выбора условий:

image::vcl/components/filter-add-condition.png[align="center"]

Рассмотрим возможные типы условий:

* *Properties* – атрибуты данной сущности и связанных с ней сущностей. Отображаются только персистентные и xref:dyn-attr:index.adoc[динамические] атрибуты, соответствующие правилам, указанным в элементе <<filter-element-properties,properties>>.

[[predefined-conditions]]
* *Predefined conditions* – условия, заданные разработчиком в элементе <<filter-element-conditions,conditions>> XML-дескриптора. Предопределенные условия изначально не добавлены ни в какие фильтры, но пользователь может выбрать их на экране *Add Condition* и добавить в <<run-time-configuration,конфигурацию во время выполнения>>.

* *Configurations* могут быть созданы во время разработки или во время выполнения (в  <<configuration,редакторе конфигураций>>).

Выбранные условия отображаются в верхней части панели фильтра. Рядом с каждым условием находится кнопка image:vcl/components/filter-remove-condition.png[], позволяющая удалить их из набора.

[[registered-filters-popup-button]]
Действия выпадающего списка кнопки *Create* соответствуют xref:vcl/components/filter-components.adoc#filter-component-registration[зарегистрированным] компонентам фильтра. По умолчанию доступны три компонента фильтра – `PropertyFilter`, `JpqlFilter`, и `GroupFilter`:

* `PropertyFilter` соответствует <<property-conditions,условиям по атрибутам>>.
* `JpqlFilter` соответствует <<jpql-conditions,JPQL-условиям>>.
* `GroupFilter` соответствует <<group-conditions,группирующим условиям>>.

image::vcl/components/filter-create-filter-button.png[align="center"]

При наличии в проекте дополнения xref:search:index.adoc[Search] становится доступным новое xref:search:search-in-ui.adoc#fts-condition-in-filter-component[full-text условие].

Таким образом, у пользователя есть возможность создавать условия фильтрации во время выполнения.

[[property-conditions]]
=== Условия по атрибутам

*Property conditions editor* позволяет пользователям настраивать атрибуты компонента xref:vcl/components/property-filter.adoc[PropertyFilter]. `PropertyFilter` связан с атрибутом сущности.

image::vcl/components/filter-property-conditions.png[align="center"]

Выберите атрибут сущности в поле *Property*.

Поле *Operation* поле позволяет выбрать xref:vcl/components/property-filter.adoc#operation[оператор] условия. Список доступных операций зависит от типа атрибута.

Используйте атрибут *Parameter name*, чтобы задать имя параметра запроса. Это имя можно использовать для введения зависимостей между компонентами фильтра в конфигурации. Если значение не задано, то имя параметра генерируется случайным образом.

В поле *Caption* можно задать собственное название условия фильтрации по атрибутам.

Флажок *Operation editable* используется, чтобы задать видимость выбора оператора. Если этот флажок установлен, поле операции позволит выбрать оператор условия во время выполнения.

Флажок *Operation caption visible* используется, чтобы задать видимость названия операции.

В поле *Default value* для условия можно установить значение по умолчанию.

[[jpql-conditions]]
=== JPQL-условия

*JPQL conditions editor* позволяет пользователям создавать условия на основе выражений JPQL.

image::vcl/components/filter-jpql-conditions.png[align="center"]

В поле *Parameter type* нужно указать класс Java параметра условия. Значение `No parameter` позволяет создать условие без параметров. Например:

image::vcl/components/filter-jpql-conditions-no-parameters.png[align="center"]

Поле *Caption* обязательно для заполнения. В нем необходимо задать имя условия, отображаемое в фильтре.

Атрибут *Parameter name* используется для задания имени связанного параметра запроса. Это имя можно использовать для введения зависимостей между компонентами фильтра в конфигурации. Если значение не задано, то имя параметра генерируется случайным образом.

Флажок *Has IN expression* используется, чтобы определить, должно ли выражение JPQL содержать выражение `IN`. В этом случае приложение будет использовать компонент xref:vcl/components/values-picker.adoc[ValuesPicker] для создания списка значений условия.

В поле *Default value* для условия можно установить значение по умолчанию.

[[jpql-conditions-join]]
В поле *Join* можно указать выражение JPQL, которое будет добавлено в секцию `from` запроса контейнера данных. Это может потребоваться для создания сложного условия по атрибуту связанной коллекции. Выражение должно начинаться с предложений `join` или `left join`. Вместо алиаса отбираемой сущности в выражении следует использовать плейсхолдер `\{E}`.

image::vcl/components/filter-jpql-conditions-join.png[align="center"]

[[jpql-conditions-where]]
Значение в поле *Where* задает выражение JPQL, которое будет добавлено в секцию `where` запроса контейнера данных. Вместо алиаса отбираемой сущности в выражении следует использовать плейсхолдер `\{E}`. Параметр условия может быть только один, и если он определен в поле *Parameter type*, обозначается символом `?`. Кроме вводимых пользователем параметров, можно использовать любое количество параметров xref:data-access:jpql-extensions.adoc#session-and-user-attributes[атрибутов сессии и пользователя], например:

[source,jpql]
----
{E}.code = ? and {E}.area = :session_area
----

[[group-conditions]]
=== Группирующие условия

*Group conditions editor* позволяет пользователям объединять несколько условий в логическую группу.

image::vcl/components/filter-group-conditions.png[align="center"]

Выпадающий список *Operation* позволяет пользователям выбрать оператора логической группы.

Поле *Caption* обязательно для заполнения. В нем необходимо задать имя условия, отображаемое в фильтре.

Флажок *Operation caption visible* используется, чтобы задать видимость имени операции.

Чтобы добавить условие, нажмите кнопку *Add*. Появится экран <<add-condition,Add condition>>.

Условия фильтрации содержатся в дереве. Пользователи могут определять порядок условий фильтрации в фильтре с помощью кнопок image:vcl/components/filter-group-conditions-move-down.png[align="center"]/image:vcl/components/filter-group-conditions-move-up.png[align="center"] и удалять и редактировать условия с помощью кнопок *Remove* и *Edit*.

[[configuration]]
== Конфигурация

Конфигурация – это набор компонентов фильтра.

`LogicalFilterComponent` является корневым элементом конфигурации.

Конфигурация не содержит ссылок на все компоненты, а только ссылку на корневой элемент, из которого могут быть получены остальные компоненты фильтра.

Конфигурации могут быть созданы во время разработки или во время выполнения (с помощью экрана <<run-time-configuration,Configuration editor>>).

[[design-time-configuration]]
=== Конфигурация времени разработки

Конфигурация, созданная во время разработки, не может быть изменена и сохранена во время выполнения. Тем не менее, пользователь может использовать действие `Copy`, которое копирует все условия из конфигурации времени разработки в конфигурацию времени выполнения.

Все конфигурации, созданные во время разработки, отображаются в выпадающем меню(<<refresh-drop-down-menu,2>>) сохраненных фильтров и конфигураций:

[[run-time-configuration]]
=== Конфигурация времени выполнения

Чтобы создавать, редактировать или удалять конфигурации во время выполнения, пользователь должен иметь разрешение безопасности <<ui-filter-modifyConfiguration,ui.filter.modifyConfiguration>>.

Чтобы открыть экран *Configuration editor*, используйте действие *Edit* всплывающей кнопки настроек фильтра(<<filter-settings-button,6>>).

image::vcl/components/filter-run-time-configuration.png[align="center"]

Поле *Configuration name* обязательно для заполнения. В нем необходимо задать имя конфигурации, отображаемое в выпадающем меню(<<refresh-drop-down-menu,2>>) сохраненных фильтров и конфигураций.

[[available-for-all-users]]
С помощью флажка *Available for all users* можно установить фильтр, доступный для всех пользователей. Этот флажок доступен только пользователям, имеющим разрешение <<ui-filter-modifyGlobalConfiguration,ui.filter.modifyGlobalConfiguration>>.

[[default-for-all-users]]
С помощью флажка *Default for all users* можно указать, будет ли созданная конфигурация автоматически выбираться при открытии пользователями экрана. Этот флажок доступен, если установлен флажок *Available for all users*.

Другие поля *Configuration editor* описаны в разделе <<group-conditions>>.

[[filter-elements]]
== Элементы Filter

Определенный в XML-дескрипторе `filter` может содержать вложенные элементы. Все они описывают условия, доступные пользователю для выбора в диалоге *Add Condition*:

[[filter-element-properties]]
=== properties

`properties` позволяет сделать доступными для выбора сразу несколько атрибутов сущности. Данный элемент может иметь следующие атрибуты:

[[filter-element-properties-include]]
* `include` - обязательный атрибут, содержит регулярное выражение, которому должно соответствовать имя атрибута сущности.

[[filter-element-properties-exclude]]
* `exclude` - содержит регулярное выражение, при соответствии которому атрибут сущности исключается из ранее включенных с помощью `include`.
* `excludeProperties` - содержит список атрибутов, разделенных запятыми, которые должны быть исключены из фильтрации. В отличие от `exclude`, этот атрибут поддерживает путь по графу сущностей для указания каждого свойства в списке. Например, `customer.name`.
* `excludeRecursively` - указывает, должны ли атрибуты, перечисленные в `excludeProperties`, быть рекурсивно исключены из полного графа сущностей. Если установлено `true`, указанный атрибут и все его вложенные атрибуты не будут использоваться в фильтре.

* Коллекции (`@OneToMany`, `@ManyToMany`).
* Коллекции.
* Атрибуты, аннотированные `@SystemLevel`.
* Атрибуты типа `byte[]`.

[[filter-element-actions]]
=== actions

`actions` – это необязательный элемент, определяющий список действий для управления фильтрами.

[[filter-default-actions]]
Фреймворк предоставляет следующие действия по умолчанию во всплывающей кнопке настройки фильтра (<<filter-settings-button,6>>) image:vcl/components/filter-settings-popup.png[align="center"]:

[[filter-save-action]]
* *Save* - реализовано с помощью `FilterSaveAction` (`type="filter_save"` в XML), сохраняет изменения в текущей конфигурации фильтра.

[[filter-save-with-values-action]]
* *Save with values* - реализовано с помощью `FilterSaveWithValuesAction` (`type="filter_saveWithValues"` в XML), сохраняет изменения в текущем фильтре, используя значения в полях условия(<<condition-fields,5>>) в качестве значений фильтра по умолчанию.

[[filter-save-as-action]]
* *Save as* - реализовано с помощью `FilterSaveAsAction` (`type="filter_saveAs"` в XML), сохраняет конфигурацию фильтра под новым именем.

[[filter-edit-action]]
* *Edit* - реализовано с помощью `FilterEditAction` (`type="filter_edit"` в XML), изменяет текущую <<run-time-configuration,конфигурацию фильтра во время выполнения>> (открывает экран *Configuration editor*). Отключено для <<design-time-configuration,конфигураций во время разработки>>.

[[filter-remove-action]]
* *Remove* - реализовано с помощью `FilterRemoveAction` (`type="filter_remove"` в XML), удаляет текущую конфигурацию фильтра во время выполнения. Отключено для конфигураций во время разработки.

[[filter-make-default-action]]
* *Make default* - реализовано с помощью `FilterMakeDefaultAction` (`type="filter_makeDefault"` в XML), делает конфигурацию фильтра конфигурацией по умолчанию для этого экрана. Фильтр будет автоматически отображен на панели фильтров при открытии экрана. Имеет приоритет над атрибутом <<design-time-configuration-default,default>> конфигурации во время разработки и флажком <<default-for-all-users,Default for all users>> конфигурации во время выполнения. Отключено, если в экран не добавлен фасет `screenSettings`.

[[filter-copy-action]]
* *Copy* - реализовано с помощью `FilterCopyAction` (`type="filter_copy"` в XML), копирует все условия из конфигурации во время разработки в конфигурацию во время выполнения.

[[filter-clear-values-action]]
* *Clear values* - реализовано с помощью `FilterClearValuesAction` (`type="filter_clearValues"` в XML), очищает значения условия фильтра(<<condition-fields,5>>).

[[filter-add-condition-action]]
* *Add* реализовано с помощью `FilterAddConditionAction` (`type="filter_addCondition"` в XML), добавляет условие к текущей конфигурации фильтра.

Пользователи могут переопределять действия <<filter-default-actions,default>> с помощью вложенного элемента `actions`. Например:

[[filter-attributes]]
== Атрибуты Filter

[[filter-attribute-auto-apply]]
=== autoApply

Атрибут `autoApply` определяет, когда применяется фильтр. Если значение атрибута `false`, фильтр будет применен только после нажатия кнопки *Refresh*. Если значение атрибута `true`, фильтр применяется сразу после изменения компонентов фильтра. Значение по умолчанию указано в свойстве xref:app-properties.adoc#jmix.ui.component.filter-auto-apply[jmix.ui.component.filter-auto-apply].

[[filter-attribute-columns-count]]
=== columnsCount
Параметр `columnsCount` задает количество столбцов, которые будут отображаться в одной строке. Значение по умолчанию указано в свойстве xref:app-properties.adoc#jmix.ui.component.filter-columns-count[jmix.ui.component.filter-columns-count].

[[filter-attribute-caption-position]]
=== captionPosition

Атрибут `captionPosition` определяет положение заголовка дочерних компонентов фильтра: `TOP` или `LEFT`. Значение по умолчанию – `LEFT`.

[[permissions]]
== Разрешения

Для расширенного управления фильтрами требуются специальные разрешения:

[[ui-filter-modifyConfiguration]]
* Для создания/изменения/удаления <<configuration,конфигураций>> фильтра пользователь должен иметь разрешение `ui.filter.modifyConfiguration`.

[[ui-filter-modifyGlobalConfiguration]]
* Для создания/изменения/удаления глобальных (<<available-for-all-users,доступных всем пользователям>>) фильтров пользователь должен иметь разрешение `ui.filter.modifyGlobalConfiguration`.

[[ui-filter-modifyJpqlCondition]]
* Для создания/изменения <<jpql-conditions,условий JPQL>> во время выполнения пользователь должен иметь разрешение `ui.filter.modifyJpqlCondition`.

Вы можете либо предоставить пользователям предопределенную ресурсную роль *UI: edit filters*, либо настроить необходимые разрешения в своей собственной xref:security:resource-roles.adoc#create[роли].
