= 查找标签
:page-aliases: backoffice-ui:actions/standard-actions/tag-lookup-action.adoc

`TagLookupAction` 是一个查找操作，通过查找界面选择 xref:data-model:entities.adoc[实体] 实例，并将该实例设置给 xref:vcl/components/tag-picker.adoc[TagPicker]。

该操作通过 `io.jmix.ui.action.tagpicker.TagLookupAction` 类实现，在 XML 中需要使用操作属性 `type="tag_lookup"` 定义。可以用 `action` 元素的 XML 属性定义通用的操作参数，参阅 xref:actions/declarative-actions.adoc[声明式操作] 了解细节。下面我们介绍 `TagLookupAction` 类特有的参数。

== 属性

下列参数可以在 XML 或 Java 中设置：

* `openMode` - 查找界面的打开模式，是 `OpenMode` 枚举类型的值：`NEW_TAB`、`DIALOG` 等。默认情况下，`TagLookupAction` 用 `THIS_TAB` 模式打开查找界面。
* `screenId` - 查找界面的字符串 id。`TagLookupAction` 默认会使用带有 `@PrimaryLookupScreen` 注解的界面，或标识符为 `<entity_name>.browse` 格式的界面，例如，`demo_Customer.browse`。
* `screenClass` - 查找界面控制器的 Java 类。比 `screenId` 有更高的优先级。
* `multiSelect` - 是否在查找界面启用多选。默认为 `true`。
