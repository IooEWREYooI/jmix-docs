= BPMN 设计器

Studio 提供 BPMN 设计器支持在 IDE 中对业务流程进行建模。仅当项目添加了 xref:bpm:index.adoc[BPM 扩展组件] 才会显示此设计器。

include::subscription-note-ent.adoc[]

image::bpm-designer.png[align="center"]

除了 Jmix Studio 中的 BPMN 设计器之外，BPM 组件本身也提供了运行时业务流 xref:bpm:modeler.adoc[建模器]。它们功能类似，可以根据需要使用任一种进行设计。

如果需要在运行的应用程序中创建新流程或对生产系统上的现有流程进行快速修改，则可使用运行时建模器。

在 Jmix Studio 中使用 BPMN 设计器的好处是，可以直接编辑位于项目源代码中的 `bpmn.xml` 流程定义文件。将项目制件交付至生产环境时，将自动部署这些文件。

另一个优点是能够使用 IDE 的功能，如代码完成，代码生成，导航等。

例如，当为用户任务元素定义 xref:bpm:jmix-screen.adoc[Jmix 界面流程表单] 时，可以很方便导航到界面类的源代码，甚至生成新界面。

image::bpm-create-form.png[align="center"]

对于流程中需要的服务、bean、用户 provider、Java 代理和监听器，都支持相同的导航行为。

如需在 Studio 的 BPMN 设计器中创建新流程，使用 xref:studio:tool-window.adoc[Jmix 工具窗口] 中的菜单。

image::bpm-new.png[align="center"]

设置流程的编码和名称。编码会作为 XML 文件名的一部分。

image::bpm-name.png[align="center"]

会在 `resources/processes` 目录创建一个新文件。这个目录的流程文件会在应用程序启动时自动部署，可以在 *BPM -> Process definitions（流程定义）* 界面进行查看。

image::bpm-processes.png[align="center"]

在 BPMN 设计器中，画布可以切换不同的查看方式，BPM Inspector 也与 xref:bpm:modeler.adoc[运行时建模器] 中的属性面板有相同的功能。
